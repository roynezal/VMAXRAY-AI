1. System Overview

The VMAX Strategy AI Chat system allows users to interact with an AI trading agent from anywhere. The chat server runs on a Windows 7 laptop (192.168.10.163). The chat UI is hosted on GitHub Pages and connects via a Google Apps Script tunnel to the Win7 server.



1.1 Architecture

Win7 Laptop (192.168.10.163)

  • vmaxray_chat_server.py — Python HTTP server on port 8080

  • Python 3.7.9 (C:\Python37\python.exe)

  • SQLite DB (C:\vmaxray-chat\vmaxray.db)

  • vmaxray_tunnel.py — GAS tunnel client

  • SSH access: Opencode / vmaxray3693 (Bitvise)

Cloud Services

  • GitHub Pages — https://roynezal.github.io/vmaxchat/

  • Google Apps Script Tunnel — relays requests to Win7

  • InfinityFree Landing — https://vmaxcrypto.infinityfree.me/



1.2 Data Flow

Boot → Win7 Task Scheduler / manual start

├─ Start vmaxray_chat_server.py :8080 (C:\Python37\python.exe)

└─ Server binds 0.0.0.0:8080 (LAN accessible)



Browser → GitHub Pages (https://roynezal.github.io/vmaxchat/)

└─ index.html loads, uses hardcoded GAS URL

└─ Sends message to GAS /chat endpoint



GAS Tunnel → Win7 tunnel client (vmaxray_tunnel.py)

└─ Forwards request to http://192.168.10.163:8080/chat



Win7 Chat Server → AI Provider (Groq/OpenRouter/Claude)

└─ Returns response back through tunnel chain

Service

3. Auto-Boot Setup

3.1 Master Boot Script — start_vmaxray.sh

Location

/home/live/VMAXRAY/start_vmaxray.sh



#!/bin/bash

LOG=~/VMAXRAY/boot.log

echo "$(date) Starting VMAXRAY..." > $LOG



# Kill old instances

pkill -9 -f vmaxray_chat_server 2>/dev/null

fuser -k 8080/tcp 2>/dev/null

pkill -f "ssh.*localhost.run" 2>/dev/null

sleep 3



# Start chat server

cd ~/VMAXRAY

python3 vmaxray_chat_server.py 8080 >> $LOG 2>&1 &

sleep 4



# Start tunnel

ssh -o StrictHostKeyChecking=no -R 80:localhost:8080 nokey@localhost.run >> $LOG 2>&1 &

sleep 8



# Capture URL and push to InfinityFree

TUNNEL_URL=$(grep -o 'https://[^ ]*\.lhr\.life' $LOG | tail -1)

echo "$TUNNEL_URL" | curl -s -T - \

"ftp://ftpupload.net/htdocs/tunnel_url.txt" \

--user "if0_41171553:S7010877a"



# Start watchdog

bash ~/VMAXRAY/watchdog.sh &



3.2 Watchdog Script — watchdog.sh

Runs in the background and checks every 2 minutes if the tunnel and chat server are still alive. Restarts and re-uploads the URL if anything drops.



Location

/home/live/VMAXRAY/watchdog.sh



Check Interval

Every 120 seconds



#!/bin/bash

LOG=~/VMAXRAY/boot.log

while true; do

if ! pgrep -f "ssh.*localhost.run" > /dev/null; then

ssh -R 80:localhost:8080 nokey@localhost.run >> $LOG 2>&1 &

sleep 8

TUNNEL_URL=$(grep -o 'https://[^ ]*.lhr.life' $LOG | tail -1)

echo "$TUNNEL_URL" | curl -s -T - \

"ftp://ftpupload.net/htdocs/tunnel_url.txt" \

--user "if0_41171553:S7010877a"

fi

if ! pgrep -f "vmaxray_chat_server" > /dev/null; then

python3 ~/VMAXRAY/vmaxray_chat_server.py 8080 >> $LOG 2>&1 &

fi

sleep 120

done

4.2 updateChatUrl() Function

JavaScript function in index.html that fetches tunnel_url.txt and updates the AI Chat card link and badge status. Runs on page load and every 60 seconds.



async function updateChatUrl() {

try {

var resp = await fetch(

'https://vmaxcrypto.infinityfree.me/tunnel_url.txt?t=' + Date.now()

);

var url = await resp.text();

url = url.trim();

if (url && url.startsWith('https://')) {

document.getElementById('chatCard').href = url + '/hermes_chat.html';

document.getElementById('aiBadge').className = 'badge badge-online';

document.getElementById('aiBadge').innerHTML = '🤖 AI Chat: ✅ Live';

}

} catch(e) {

document.getElementById('aiBadge').innerHTML = '🤖 AI Chat: Offline';

}

}

updateChatUrl();

setInterval(updateChatUrl, 60000);

5.5 Win7 AI Chat Migration — Tunnel Architecture

5.5.1 How It Works (April 5, 2026 — GAS GitHub Relay)

The AI Chat server runs on Windows 7 (192.168.10.163). The chat UI is hosted on GitHub Pages. A Google Apps Script (GAS) deployed on InfinityFree acts as a relay: it forwards requests from the web UI to the Win7 server and returns the responses. No SSH tunnel (plink/localhost.run) is required. The system is fully self-contained: Win7 server + GAS proxy + GitHub Pages UI.



┌─────────────────────────────────────────────────────────────┐

│ User Browser (GitHub Pages) │

│ ↓ loads https://roynezal.github.io/vmaxchat/index.html │

│ ↓ sends chat message to GAS endpoint │

│ POST https://vmaxcrypto.infinityfree.me/tunnel.php │

└──────────────────────┬──────────────────────────────────────┘

│

▼

┌─────────────────────────────────────────────────────────────┐

│ InfinityFree (GAS Relay) │

│ tunnel.php forwards request to Win7 server │

└──────────────────────┬──────────────────────────────────────┘

▼ (HTTP POST /chat)

┌─────────────────────────────────────────────────────────────┐

│ Win7 Laptop (192.168.10.163) │

│ Server: C:\Python37\python.exe vmaxray_chat_server.py │

│ Response → GAS → Browser │

└─────────────────────────────────────────────────────────────┘



5.5.2 GAS Relay Configuration (InfinityFree)

The Google Apps Script (GAS) is deployed on InfinityFree as tunnel.php. It accepts POST requests from the GitHub Pages chat UI and forwards them to the Win7 server at http://192.168.10.163:8080/chat. The GAS returns the AI response directly to the browser. CORS headers are set to allow GitHub Pages origin. No authentication is required on the local network.



# Win7 tunnel command (plink.exe 0.83 32-bit)

plink.exe -ssh -R 80:127.0.0.1:8080 -N ^

-hostkey SHA256:FV8IMJ4IYjYUTnd6on7PqbRjaZf4c1EhhEBgeUdE94I ^

-batch nokey@localhost.run



5.5.3 Endpoint Flow

The chat UI (index.html on GitHub Pages) sends user messages directly to the GAS endpoint: https://vmaxcrypto.infinityfree.me/tunnel.php. The GAS forwards the request to the Win7 server's /chat endpoint. The Win7 server must be running and accessible on the local network at 192.168.10.163:8080. No public tunnel URL is needed; the GAS acts as the public-facing proxy.

5.5.5 Critical Rules

1. NEVER hardcode IP addresses in index.html — the landing page must always use tunnel_url.txt to get the chat URL dynamically.

2. The tunnel runs from MiniOS, NOT from Win7 — Win7 has no SSH client. The command is -R 80:192.168.10.163:8080 (forward to Win7).

3. Always backup index-good.html before changes — this is the known-good version of the landing page.

4. Google search before any fix — this is in AGENTS.md as a mandatory rule.
